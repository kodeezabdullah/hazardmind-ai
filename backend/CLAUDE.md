## API Endpoints
POST /analyze      -> Start pipeline
GET  /status/{id}  -> Check progress
GET  /results/{id} -> Final results
GET  /band-log/{id}-> Agent discussion log
GET  /health       -> Health check

## Pipeline Flow
User -> POST /analyze
     -> event_id = UUID4()
     -> INSERT disaster_events
     -> Band @hazardmind-satellite
     -> Return {job_id: event_id}

## Progress Steps
1. received  -> POST called
2. satellite -> Agent 1 processing
3. hazard    -> Agent 2 processing
4. impact    -> Agent 3 processing
5. report    -> Agent 4 processing
6. complete  -> All done
7. failed    -> Error

## Database Tables
disaster_events   <- Backend writes
satellite_results <- Agent 1 writes
hazard_zones      <- Agent 2 writes
impact_data       <- Agent 3 writes
final_reports     <- Agent 4 writes
